在jQuery库中，几乎所有的插件都被限制在他的命名空间里，通常，全局对象都被很好地存储在jQuery命名空间里面，因此当把jQuery和其他库一起使用时，不会出现冲突

1)jQuery库在其他库之后导入

在其他库和jQuery库都导入完毕后，可以在任何时候调用jQuery.noConflict()函数来将变量$的控制权移交给其他库。示例如下：

‹ script type="text/javascript" src=" Prototype.js"›//引入Prototype库‹/script›

‹ script type="text/javascript" src=" jquery-2.2.0.js"›//引入jQuery库‹/script›

‹script›

jQuery.noConflict(); //将变量$的控制权移交给Prototype库

jQuery(function(){

  jQuery("p").click(function(){

    alert("111");

  })

})

$("pp").style.display=none; //使用prototype

‹/script›


此外，还有另一种选择，如果想确保jQuery库不会和其他库冲突，又想自定义一种快捷方式，可以用如下操作

var $j=jQuery.noConflict();

$j(function(){

  $j("p").click(function(){

    alert("111");

  })

})

$("pp").style.display=none; //使用prototype



2)jQuery库在其他库之前导入

如果是jQuery库在其他库之前就导入了，那么可以直接使用jQuery来做一些jQuery的工作，同时，可以用$()方法作为其他库的快捷方式，无需调用jQuery.noConflict()函数
